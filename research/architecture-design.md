# パラレルワールド・ソーシャルシミュレーター — アーキテクチャ設計

## コンセプト

**「現実を食べて、未来を走る並行世界」**

現実世界のニュースをリアルタイムに取り込みながら、シミュレーション内の時間を加速させ、
常に数ヶ月先の「ありえた未来」を生成し続ける。現実が追いついたとき、
シミュレーションの予測と現実の差分が可視化される。

```
現実世界の時間軸:  ─────●────────────────────────────→
                     2026/03  (今ここ)

シミュレーション:   ─────●━━━━━━━━━━━━━━━━━━━━━━━━━●→
                     2026/03                    2026/09
                     (現実と同期)         (6ヶ月先を走行中)

                     ┃ 新ニュース投入
                     ┃ ↓ 分岐点が生まれる
                     ●━━━━━━━━━→ 分岐A（楽観シナリオ）
                     ┗━━━━━━━━━→ 分岐B（悲観シナリオ）
```

## コアメカニクス

### 1. 時間加速エンジン (Time Warp Engine)

現実の1日 = シミュレーション内の約1〜2週間（調整可能）

```
リアルタイム          シミュレーション時間
─────────           ──────────────
1時間        →       1〜2日
1日          →       1〜2週間
1週間        →       1〜3ヶ月
```

- シミュレーションは24時間常時稼働
- 各タイムステップで全エージェントが行動（投稿・反応・意見更新）
- 時間加速率はトピックの「温度」（議論の活発さ）で動的に変化

### 2. ニュースインジェクション (Reality Anchor)

現実世界のニュースがシミュレーションの「アンカーポイント」になる。

```
[現実のニュース] → [トピック抽出・要約] → [シミュレーションに注入]
                                              │
                                         エージェントが反応
                                              │
                                    加速された時間の中で議論が発展
                                              │
                                       数ヶ月先の世論分布を予測
```

- 新しいニュースが入るたびに「分岐点」が生まれる
- 過去のシミュレーション予測と現実の差分を記録（精度追跡）

### 3. エージェント設計

各エージェントは「架空の市民」として以下を持つ：

```python
Agent:
  # アイデンティティ（固定）
  personality:     # ビッグファイブ性格特性
  demographics:    # 年齢層、職業、地域（架空）
  values:          # コア価値観（自由、安全、平等、伝統、革新...）

  # 認知バイアス（固定〜緩やかに変化）
  confirmation_bias:    # 確認バイアスの強さ (0.0 - 1.0)
  authority_bias:       # 権威バイアス
  bandwagon_effect:     # 同調圧力への感受性

  # 動的状態
  opinions:        # トピックごとの意見 (-1.0 〜 +1.0)
  memory:          # 最近の議論の記憶（要約ベース）
  relationships:   # 他エージェントとの関係性スコア
  trust_scores:    # 情報源への信頼度
  emotional_state: # 感情状態（怒り、不安、希望...）
```

### 4. ソーシャルネットワーク構造

```
[Small World Network]
     ○─○─○
    /│   │\
   ○ │ ○ │ ○     ← 密なローカルクラスタ
    \│/ \│/
     ○───○        ← 弱い紐帯（weak ties）でクラスタ間接続
    /│   │\
   ○ │ ○ │ ○
    \│   │/
     ○─○─○
```

- 初期構造: Watts-Strogatzスモールワールド or Barabási-Albertスケールフリー
- 動的再配線: 意見が近いエージェント同士は接続が強化（ホモフィリー）
- エコーチャンバー形成が自然に発生

### 5. 議論サイクル（1タイムステップの流れ）

```
┌─────────────────────────────────────────┐
│  1. 情報取得                              │
│     エージェントがフィードからコンテンツを受け取る  │
│     (レコメンドアルゴリズム + ネットワーク経由)    │
├─────────────────────────────────────────┤
│  2. 認知処理                              │
│     LLMが性格・バイアス・記憶に基づいて         │
│     情報を解釈し、感情反応を生成               │
├─────────────────────────────────────────┤
│  3. 行動決定                              │
│     - 投稿する / しない                     │
│     - 誰かに返信する                        │
│     - 意見を変える / 強化する                │
│     - フォロー / アンフォロー                 │
├─────────────────────────────────────────┤
│  4. 意見更新                              │
│     議論の結果に基づいて意見スコアを更新         │
│     記憶を要約して保存                       │
├─────────────────────────────────────────┤
│  5. ネットワーク更新                         │
│     関係性スコアの変化                       │
│     新しい接続 / 切断                       │
└─────────────────────────────────────────┘
```

## パラレルワールドの分岐メカニズム

### タイムライン管理

```
                    ニュースA投入
                        │
    ────────────────────●────────────────────→ Main Timeline
                        │
                        ├───────────→ Fork 1: "もしニュースAが起きなかったら"
                        │
                        └───────────→ Fork 2: "もし逆の展開だったら"

                              ニュースB投入
                                  │
    ──────────────────────────────●──────────→ Main Timeline (更新)
                                  │
                                  └─────────→ Fork 3: 新たな分岐
```

- **メインタイムライン**: 現実のニュースをすべて反映した「最も確からしい未来」
- **フォーク**: 特定のイベントが「起きなかった」or「別の展開だった」場合の並行世界
- フォークは最大N本（リソース制約）、古いものは凍結・アーカイブ

### 現実との照合 (Reality Diff)

```
時間の経過:

  シミュレーション予測 (2026/01に生成):
    "2026/06 → AI規制法案は可決、世論は60:40で賛成"

  現実 (2026/06に到達):
    "AI規制法案は修正の上可決、世論は55:45で賛成"

  Reality Diff:
    予測精度: 82%
    主な乖離: 修正条項の影響を過小評価
```

### 技術選定

| コンポーネント | 技術 | 理由 |
|-------------|------|------|
| シミュレーションコア | Python | エージェントベースモデリングのエコシステム |
| LLM | Claude API (claude-sonnet-4-6) | コスト効率、日本語対応、多様な性格ロールプレイ |
| ネットワーク分析 | NetworkX | グラフ操作・分析の標準 |
| データベース | SQLite → PostgreSQL | 軽量スタート、スケール時に移行 |
| APIサーバー | FastAPI | WebSocket対応、非同期処理 |
| フロントエンド | Next.js + D3.js | リアルタイム可視化 |
| タスクスケジューラ | APScheduler or Celery | 定期的なニュース取得・シミュレーションステップ |
| ニュース取得 | NewsAPI + RSS | 多様なソースからのリアルタイム取得 |

## エージェント規模とコスト試算

### 想定規模
- **エージェント数**: 100〜200体（初期）
- **タイムステップ**: 1ステップ = シミュレーション内1日
- **1日あたりのステップ数**: 7〜14ステップ（1〜2週間分/現実1日）
- **アクティブエージェント率**: 各ステップで30〜50%が行動

### LLMコール試算（1現実日あたり）
```
1ステップあたり:
  アクティブエージェント: 60体 (平均)
  × LLMコール/エージェント: 1.5回 (認知処理 + 投稿生成)
  = 90コール/ステップ

1日あたり:
  90コール × 10ステップ = 900コール/日

  入力トークン: ~800 tokens/コール (システムプロンプト + 記憶 + コンテキスト)
  出力トークン: ~200 tokens/コール (投稿 + 意見更新)

  日次コスト概算 (Claude Sonnet):
  入力: 900 × 800 = 720K tokens → ~$2.16
  出力: 900 × 200 = 180K tokens → ~$2.70
  合計: ~$5/日 → ~$150/月
```

### 最適化戦略
- **バッチ処理**: 複数エージェントの処理を1コールにまとめる
- **キャッシュ**: 類似コンテキストの応答をキャッシュ
- **軽量判断**: 意見更新のみの場合はルールベースで処理（LLM不要）
- **重要度フィルタ**: 議論の「温度」が低いエージェントはスキップ

## 初期トピック候補（2026年3月時点）

シミュレーションの初期トピックとして、以下の社会課題が適切：

1. **AI規制・ガバナンス** — 世界的に議論が活発、数ヶ月で大きな動きが予想される
2. **気候変動政策** — 長期的な議論、意見が分かれやすい
3. **労働と自動化** — AI/ロボティクスによる雇用への影響
4. **デジタルプライバシー** — 監視技術 vs 安全保障
5. **都市計画・住宅問題** — ローカルかつ普遍的な課題

## フェーズ分け

### Phase 1: プロトタイプ（MVP）
- 50エージェント、単一トピック
- 時間加速エンジン（基本）
- ニュース手動投入
- CLI出力 + シンプルなWeb可視化
- 分岐なし（メインタイムラインのみ）

### Phase 2: コア機能
- 100〜200エージェント、複数トピック
- ニュース自動取得（NewsAPI連携）
- タイムライン分岐機能
- リアルタイムWeb可視化
- Reality Diff基本機能

### Phase 3: 作品としての完成
- フル可視化（ネットワーク3D、タイムライン分岐マップ）
- インタラクティブ介入（観客がイベントを注入）
- Reality Diffダッシュボード
- 長期稼働の安定性
